Pipeliner User Guide

Visual pipeline editor for orchestrating AI agent workflows. Part of the Monks.Flow platform.

Overview

Pipeliner is a drag-and-drop canvas for building creative production workflows. You chain specialized AI agents together -- briefing, strategy, content creation, quality scoring, and delivery -- into an executable pipeline. Each agent has typed input and output ports; connecting them defines the data flow.

The application has two primary views:

view_list

Workflow List

Browse, create, run, and delete your saved workflows. This is the landing page after sign-in.

account_tree

Pipeline Editor

The full canvas where you drag agents, wire connections, configure nodes, and execute runs.

Workflow List

The workflow list page is your home screen. It displays all saved workflows as cards in a responsive grid.

Creating a Workflow

Click the New Workflow button in the page header, or click the dashed "+ New Workflow" card in the grid. A new workflow named "Untitled Workflow" is created and the editor opens immediately.

Workflow Cards

Each workflow card shows:

  • Name and description -- the workflow title and an optional description.
  • Node count -- how many agent nodes the workflow contains.
  • Last modified date -- when the workflow was last updated.
  • Color bar -- a rotating color accent (blue, orange, green, purple, red) for visual distinction.

Card Actions

Each workflow card provides three actions:

  • play_arrow Run -- Start a pipeline execution directly from the list. A snackbar confirms "Workflow run started".
  • edit Edit -- Open the workflow in the pipeline editor. You can also click anywhere on the card body.
  • delete_outline Delete -- Remove the workflow permanently. A snackbar confirms "Workflow deleted".

Empty State

When no workflows exist, the page shows an illustrated empty state with a prompt to create your first workflow.

Editor Toolbar

The pipeline editor has a glass-style toolbar across the top with three zones:

Left Zone

  • arrow_back Back button -- Returns to the workflow list.
  • Brand mark -- "Pipeliner" logo with a "Monks.Flow" badge.
  • Workflow name -- An inline-editable text field. Click to rename; the name saves on blur.

Center Zone

  • hub Node count -- Live count of nodes currently on the canvas.
  • timeline Connection count -- Live count of wires between nodes.

Right Zone

  • fit_screen Fit to Screen -- Centers and scales the canvas to show all nodes.
  • save Save -- Persists the current pipeline state (nodes, edges, positions, canvas viewport).
  • play_arrow Run -- Saves the pipeline and then starts execution. Disabled when the canvas has no nodes.

Searching Agents

The search bar at the top of the palette filters agents in real-time by name or description. The filter applies across all groups simultaneously.

Collapsible Groups

Click a group header to expand or collapse that section. The I/O and Creative groups are expanded by default; the Operational group starts collapsed. Each header shows a badge with the agent count for that group.

Palette Items

Each agent entry in the palette shows:

  • A color-coded icon matching its group
  • The agent name and a brief description (truncated to 2 lines)
  • Input/output port count badges (e.g., "2 in", "3 out")

A footer at the bottom reminds you: "Drag agents onto canvas to build workflow."

Adding Nodes to the Canvas

Drag any agent from the palette onto the canvas to create a node. The node appears at the drop position with input ports on the left, output ports on the right, and a colored header matching the agent's group.

When the canvas is empty, a centered placeholder shows the Pipeliner logo with three hints: "Drag to add", "Connect ports", and "Configure agents".

Connecting Nodes

There are two ways to wire nodes together:

drag_indicator

Method 1: Drag-to-Connect

1

Click and hold on an output port (right side of a node).

2

Drag the wire to an input port (left side of another node).

3

Release to create the connection. A bezier curve wire appears between the ports.

ads_click

Method 2: Click-to-Connect

1

Double-click a port to "arm" it. The port enlarges and shows a pulsing glow animation to indicate it is armed.

2

All compatible ports on other nodes light up with a breathing glow animation, indicating they are valid targets. Ports on the same node do not light up (no self-connections).

3

Single-click any highlighted port to complete the wire.

?

Press Esc or click empty canvas space to cancel and disarm the port.

In click-to-connect mode, the canvas enters a special connect mode where node headers and config controls dim and become non-interactive, focusing attention on the ports.

Duplicate connections between the same two ports are prevented. If you try to create a wire that already exists, the armed port is simply disarmed.

Node Configuration

Each node on the canvas can be configured after placement:

  • Change agent type: Use the Agent dropdown below the node header to switch to a different agent. When you change the agent type, all existing connections to the node are removed (since the ports change) and the node's icon, color, and ports update to match the new agent.
  • Rename the workflow: Edit the workflow name directly in the toolbar's inline text input.
  • Delete a node: Hover over the node to reveal the X button in the header's top-right corner. Clicking it removes the node and all its connections.
  • Move a node: Drag the colored header bar to reposition the node. Foblex provides snap-to-grid alignment lines during dragging.

Google Drive Node

The Google Drive node operates in two modes depending on how it is wired:

cloud_download

Pull Mode (Import)

When the Google Drive node's output port (text_content) is connected downstream, the node imports a file from Google Drive. The backend calls the Drive API with the user's OAuth access token, downloads the file, and extracts the text content. Supported formats include Google Docs, PDFs, DOCX, and plain text files.

cloud_upload

Push Mode (Export)

When the Google Drive node's input port (content) receives data from upstream agents, the node exports results to a Google Doc in the user's Drive. The pipeline output is formatted and written as a document.

The Google Drive node uses the same OAuth token from your Google sign-in. Files are imported from and exported to your personal Google Drive.

Saving & Running

save

Save

Click the Save button in the toolbar to persist the pipeline to the database. The full graph state is serialized: node positions, agent configurations, edge connections, canvas position, and zoom scale. A snackbar confirms "Workflow saved".

play_arrow

Run

Click Run to execute the pipeline. The workflow is automatically saved first, then execution begins. Nodes execute in topological order -- each agent waits for its input data before running. A snackbar confirms the run status.

You can also run a workflow directly from the workflow list page without opening the editor -- click the Run button on any workflow card.

Import from CoreAgents

Pipeliner integrates with the CoreAgents platform. When you use "Send to Pipeline" in a CoreAgents session, the browser opens Pipeliner with special query parameters:

Manifest Handoff Flow

1

CoreAgents redirects to Pipeliner with source=core-agents, session_id=..., and manifest=ready in the URL.

2

Pipeliner detects these parameters and shows a confirmation dialog asking: "A manifest is ready from CoreAgents session. Import it to create a new workflow pre-configured with agent outputs?"

3

Click Import to proceed. Pipeliner fetches the manifest from the CoreAgents API, creates a new workflow, and opens the editor with nodes pre-populated.

4

Click Cancel to dismiss. The query parameters are cleared either way.

What the Manifest Contains

A CoreAgents manifest includes:

  • Session ID and title -- used to name the imported workflow.
  • Agent outputs -- key-value pairs of each agent's generated content from the session.
  • Grounding documents -- uploaded files with URIs, signed URLs, MIME types, and optionally extracted text content.
  • Conversations -- chat history from the CoreAgents session (optional).
  • Timestamps -- creation and last-updated times.

After import, you can reconfigure, add, or remove nodes before running the pipeline.

Grounding Context

Grounding context refers to documents and files that provide real-world data for agents to reference during execution.

  • Upload files -- Use the backend's /io/upload endpoint to upload local files (PDF, DOCX, images, video). Uploaded files are parsed and their text content is extracted for agent consumption.
  • Import from Google Drive -- Use the /io/import-drive endpoint or wire a Google Drive node in pull mode to bring Drive files into the pipeline as grounding documents.
  • Flow into agents -- Grounding docs are injected into the agent execution context. When imported from a CoreAgents manifest, the grounding documents carry their extracted text content, signed URLs, and metadata so agents can reference the source material.
Grounding documents give agents factual context -- brand guidelines, campaign briefs, competitive analyses -- that grounds their output in real data rather than relying solely on their training.

Canvas Navigation

The pipeline canvas uses the Foblex Flow library for an interactive, zoomable, pannable workspace.

  • Zoom: Scroll the mouse wheel to zoom in and out. Zoom range is 0.3x to 2.0x.
  • Pan: Click and drag on empty canvas space to pan the viewport.
  • Fit to Screen: Click the fit_screen button in the toolbar to auto-center and scale the canvas so all nodes are visible (with 50px padding).
  • Selection area: Drag on the canvas to create a rectangular selection box to select multiple nodes at once.
  • Alignment guides: When dragging nodes, alignment lines appear to help you snap nodes to grid positions.
  • Dot-grid background: The canvas shows a subtle circle-pattern grid that moves with zoom and pan for spatial orientation.

Theme Toggle

Pipeliner supports three theme modes, toggled by clicking the theme button in the top bar of the shell (the application frame visible on all pages).

light_mode

Light

White backgrounds, dark text. Default for most users.

dark_mode

Dark

Dark backgrounds, light text. Sets data-theme="dark" on the document.

routine

System

Follows your operating system's light/dark preference automatically.

The toggle cycles through Light → Dark → System → Light. Your preference is saved to localStorage under the key pipeliner-theme and persists across sessions. In System mode, the theme reacts live to OS-level dark mode changes.

Ops Console Link

The top bar of the application shell includes an Ops Console link in the upper right, next to the theme toggle button. Clicking it opens the Ops Console application (https://opsagent.mf4g.studio) in a new browser tab. This provides quick navigation between Pipeliner and the broader Monks.Flow operations platform.

The link shows a hub icon, the text "Ops Console", and a small open_in_new external link indicator.
